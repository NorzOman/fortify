Getting Started

To use the Fortify API, you'll need:

  • An API key (get one by signing up)
  • Basic understanding of REST APIs
  • A tool to make HTTP requests (curl, Postman, etc.)

Authentication

Get API Token

GET /get_token 
curl -X GET https://vault-7-rebooted.vercel.app/get_token

Returns a token valid for 1 hour

API Endpoints

File Scan

POST /file_scan 
curl -X POST https://vault-7-rebooted.vercel.app/file_scan \
-H "Authorization: Bearer {token}" \
-H "Content-Type: multipart/form-data" \
-F "file=@/path/to/file"

Parameters

file Required. The file to scan

Response

{
  "scan_id": "abc123",
  "status": "completed",
  "results": {
    "malicious": false,
    "score": 0,
    "detections": []
  }
}

URL Scan

POST /url_scan 
curl -X POST https://vault-7-rebooted.vercel.app/url_scan \
-H "Authorization: Bearer {token}" \
-H "Content-Type: application/json" \
-d '{"url": "https://example.com"}'

Parameters

url Required. The URL to scan

Response

{
  "domain_info": {
    "registrar": "Example Registrar",
    "created_date": "2020-01-01"
  },
  "risk_score": 2,
  "risk_level": "SAFE",
  "security_issues": []
}

Rate Limits

Free tier 100 requests/day
Pro tier 1000 requests/day

Error Handling

400 Bad Request - Invalid parameters
401 Unauthorized - Invalid token
429 Too Many Requests - Rate limit exceeded
500 Internal Server Error
